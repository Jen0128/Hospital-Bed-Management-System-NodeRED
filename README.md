# 🏥 醫院智慧病床管理系統 (Hospital Bed Management System)

本專案是一款基於 **Node-RED** 視覺化開發工具與 **MySQL** 資料庫所建構的「智慧病床管理系統」。系統專為醫護人員（如護理人員）設計，旨在將傳統繁瑣的人工查床、手動登記流程數位化與智能化，提供即時的床位監控、動態病床媒合分配、智慧風險警報以及多維度數據統計報表。

---

## 🎯 開發動機 (Motivation)
在醫療機構中，病床管理是影響醫療效率與病患體驗的關鍵環節。然而，傳統的病床管理方式往往依賴人工查詢與手動分配，不僅耗時且容易出錯，導致醫療資源調度不佳、病患等待時間延長，甚至影響醫療品質。

因此，本系統希望透過數位化與智能化技術，解決傳統管理方式的痛點，達到以下三大核心目標：
* **高效分配**：自動化搜尋與動態分配空床，縮短病患等床時間。
* **便捷操作**：直觀的圖形化介面，降低醫護人員的資訊系統操作負擔。
* **智慧管理**：整合即時警報與多維度統計，輔助決策者進行智慧資源規劃。

---

## 💻 系統簡介 (Features)
本系統主要提供四大功能模組，全面優化病房營運流程：
1. **病床狀態監控**：以清晰的顏色代碼（可用、佔用、待清潔、維修）即時顯示全院病床狀態，支援快速篩選。
2. **病人管理模組**：提供電子表單記錄入院病患資訊，並結合後端邏輯自動媒合對應科室的可用空床。
3. **智慧警報機制**：定時偵測病床佔用率，當急診、ICU 賸餘床位過低或科室即時滿床時，自動觸發視覺化警報。
4. **動態統計報表**：將資料庫數據視覺化，提供動態折線圖、條形圖與儀表盤，分析病床佔用率趨勢及患者流動。

## 🔄 系統核心流程 (Workflow)

### 1. 病床監控與狀態變更流程
護理人員進入監控介面 ➔ 可選擇科室/狀態篩選 ➔ 系統儲存條件並查詢 MySQL 資料庫 ➔ 呈現篩選結果。
點擊特定病床可檢視詳情，並可手動變更狀態（如：出院轉為待清潔、打掃完畢轉為可用、報修轉為維修中），系統同步更新至資料庫。

### 2. 病人入院管理與病床分配流程
護理人員於表單填寫病患基本資料（病歷號、姓名、年齡、性別） ➔ 暫存資料並選擇主治醫師與醫療科別 ➔ 送出登記。
後端系統接收請求後，自動執行 SQL 指令尋找該科別狀態為「可用」的空床，隨機/動態指派床位後，將病患資料寫入 `patients` 資料表，並將 `beds` 狀態同步更新為「佔用」。

### 3. 統計報表生成流程
系統定時或由護理人員點擊手動刷新 ➔ 觸發後端同時向 `beds` 與 `patients` 資料表撈取大量數據 ➔ 透過 Function 節點進行聚合運算（計算佔用率、各週入出院人數、各科別分佈） ➔ 派發至多組前端 UI 圖表節點渲染呈現。

## 🛠️ Node-RED 程式架構 (Node-RED Implementation)
本專案完整落實了低代碼（Low-Code）的工作流整合，專案結構主要包含以下三個 Flow 標籤頁：

### 1. 病床監控與警報核心邏輯 (`Flow 1: 病床監控`)
* **資料同步**：透過 `系統啟動觸發器 (inject)` 達到開機自動載入最新病床狀態，並利用 `ui_toast` 發送啟動通知。
* **動態查詢**：串接 `ui_dropdown` 篩選器，並透過 `構建查詢 (function)` 節點動態生成 SQL 語句送入 `mysql` 節點執行。
* **智慧警報**：內建 `定時檢查容量` 節點（每 60 秒自動執行一次），監控全院各科別佔用率，當 ICU 床位 ≤ 1 或急診床位 ≤ 2 時自動生成 `high` 等級緊急警報，並由 `ui_toast` 彈出警告。

### 2. 病患表單與床位自動分配 (`Flow 2: 病人管理`)
* **資料防錯**：表單 `ui_form` 收集資料後，由 `準備病患資料 (function)` 進行欄位完整性驗證，若資料不齊全則即時攔截並彈窗提醒。
* **事務處理 (Transaction)**：查空床與更新狀態一體化。若該科別已滿床，系統亦能智慧化處理，將病患存入資料庫但標記為「暫無床位」並發出警告通知。

### 3. 視覺化大螢幕與分析報表 (`Flow 3: 統計報表`)
* 整合 `ui_chart`、`ui_gauge` 等豐富的 Dashboard 元件。
* 後端利用進階 JavaScript Function，動態計算「平均住院天數」、「最長/最短住院天數」，並將複雜資料拆解派送至 9 個不同科室的專屬佔用率儀表盤。

## 🚀 環境架設與部署指南 (Deployment)

若要本地端或伺服器端運行本系統，請參考以下步驟：

### 預備環境 (Prerequisites)
1. **Node.js** (建議 v18 以上版本)
2. **Node-RED**
3. **MySQL Database**
4. **Node-RED 擴充節點** (需於 Manage Palette 安裝)：
   * `node-red-dashboard` (前端介面)
   * `node-red-node-mysql` (資料庫連線)

### 資料庫配置 (Database Setup)
請在你的 MySQL 資料庫中建立名為 `hospital_bed_management` 的 Database，並確保建立以下結構之資料表：
* `beds` 資料表：包含 `bed_id`, `bed_code`, `department`, `floor`, `room_number`, `bed_number`, `status`, `patient_id`, `last_cleaned` 等欄位。
* `patients` 資料表：包含 `patient_id`, `id_number`, `name`, `age`, `gender`, `department`, `doctor`, `registration_date`, `bed_id` 等欄位。

### 匯入與部署
1. 複製本專案的 `病床管理系統Node-RED.json` 原始代碼。
2. 開啟 Node-RED 編輯器介面 (`http://localhost:1880`)。
3. 點選右上角選單 ➔ **Import** ➔ 貼上 JSON 代碼並點選匯入。
4. 雙擊修改所有的 **MySQL database** 連線節點，配置你正確的 MySQL Host IP、帳號與密碼。
5. 點擊右上角紅色 **Deploy** 按鈕部署。
6. 開啟前端操作介面網址：`http://localhost:1880/ui/`，即可開始使用醫院病床管理系統！
